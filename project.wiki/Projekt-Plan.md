# Backgrund

Första iden var att systemet skulle bara hantera och spara buss biljetter vilket sedan växte till ett system som kan hantera hela verksamheten för ett företag som erbjuder buss transporter. Systemet är uppdelad i tre delar en client där kunder kan boka biljetter, en server som är en databas för biljetterna och en desktop applikation för företaget. 

# Syfte

Syftet med denna projekt är att skapa ett system som kan hantera den dagliga verksamheten på ett företag som sköter kollektiv trafik och för att tillhandahålla information om företagets resurser som: Bussar, biljetter, chaufförer, linje trafik. 

# Mål

## Kunder

Kunder ska kunna boka biljetter samt få en biljett i utskrivs format

## Buss företag

Företaget ska få en översyn av deras verksamhet detta innebär att de ska kunna
* Överblick med alla Linjer som är i drift samt att kunna redigera de.
* Skapa nya linjer eller ta bort bort befintliga.
* Se alla bussar som är i drift och inte i drift. 
* Att kunna ta bort/lägga till nya bussar samt att redigera informationen.
* Ha en lista med alla chaufförer samt att kunna lägga till/ändra/ta bort. 
* Överblick av den nuvarande tillståndet av en specifik buss/linje dvs. hur många platser som är bokade var bussen befinner sig. 
* Ett on-board system för bussen.
